# Dialog

`Dialog` renders content in an overlay that blocks interaction with the rest of the page. Built on Radix UI's dialog primitive with focus trapping and scroll locking.

## Import

```tsx
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogDescription,
  DialogFooter,
  DialogClose,
} from '@clidey/ux';
```

## Basic dialog

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="outline">Open dialog</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Edit profile</DialogTitle>
      <DialogDescription>
        Make changes to your profile here. Click save when you're done.
      </DialogDescription>
    </DialogHeader>
    <div className="space-y-4 py-4">
      <div className="space-y-1">
        <Label htmlFor="name">Name</Label>
        <Input id="name" defaultValue="Jane Doe" />
      </div>
      <div className="space-y-1">
        <Label htmlFor="username">Username</Label>
        <Input id="username" defaultValue="@janedoe" />
      </div>
    </div>
    <DialogFooter>
      <DialogClose asChild>
        <Button variant="outline">Cancel</Button>
      </DialogClose>
      <Button>Save changes</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

## Controlled dialog

```tsx
import { useState } from 'react';

function ControlledDialog() {
  const [open, setOpen] = useState(false);

  async function handleSave() {
    await saveData();
    setOpen(false);
  }

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>Open</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Confirm action</DialogTitle>
          <DialogDescription>
            This will apply changes to all selected items.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outline" onClick={() => setOpen(false)}>Cancel</Button>
          <Button onClick={handleSave}>Confirm</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Without close button

Pass `showCloseButton={false}` to hide the default X button in the corner:

```tsx
<DialogContent showCloseButton={false}>
  <DialogHeader>
    <DialogTitle>Processing...</DialogTitle>
  </DialogHeader>
  <div className="flex justify-center py-4">
    <Spinner size="lg" />
  </div>
</DialogContent>
```

## Large content dialog

```tsx
<DialogContent className="max-w-2xl">
  <DialogHeader>
    <DialogTitle>Terms of Service</DialogTitle>
  </DialogHeader>
  <ScrollArea className="h-64">
    <div className="space-y-4 text-sm">
      {/* long content */}
    </div>
  </ScrollArea>
  <DialogFooter>
    <DialogClose asChild>
      <Button variant="outline">Decline</Button>
    </DialogClose>
    <Button>Accept</Button>
  </DialogFooter>
</DialogContent>
```

## Sub-components

| Component | Description |
| --- | --- |
| `Dialog` | Root component. Manages open state. |
| `DialogTrigger` | Button or element that opens the dialog. Use `asChild` to render as your own element. |
| `DialogPortal` | Renders dialog outside the DOM tree. Used internally. |
| `DialogOverlay` | Semi-transparent backdrop. Used internally. |
| `DialogContent` | The dialog panel itself. Contains all visible content. |
| `DialogHeader` | Top section with consistent spacing for title/description. |
| `DialogTitle` | Accessible dialog title (required for a11y). |
| `DialogDescription` | Supporting text below the title. |
| `DialogFooter` | Bottom section, flex row right-aligned for action buttons. |
| `DialogClose` | Button that closes the dialog. Use `asChild` to render as any element. |

### DialogContent Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `showCloseButton` | `boolean` | `true` | Shows or hides the default close (X) button in the top-right corner. |
| `className` | `string` | — | Additional CSS classes. Use to change max-width. |
| `onOpenAutoFocus` | `(e: Event) => void` | — | Override default focus behavior on open. |
