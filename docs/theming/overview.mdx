# Theming Overview

Clidey UX uses **CSS custom properties** (variables) for all colors and design tokens. The `ThemeProvider` manages which set of variables is applied based on the active theme (light, dark, or system).

## ThemeProvider

Wrap your application root with `ThemeProvider` to enable theme management:

```tsx
import { ThemeProvider } from '@clidey/ux';

function Root() {
  return (
    <ThemeProvider defaultTheme="system" storageKey="my-app-theme">
      <App />
    </ThemeProvider>
  );
}
```

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `defaultTheme` | `"light" \| "dark" \| "system"` | `"system"` | Theme applied on first load before any user preference is stored. |
| `storageKey` | `string` | `"@clidey/ux/theme"` | `localStorage` key used to persist the user's choice. |
| `children` | `ReactNode` | — | Your application tree. |

### useTheme hook

Any component inside `ThemeProvider` can read and change the active theme:

```tsx
import { useTheme } from '@clidey/ux';

function MyComponent() {
  const { theme, setTheme } = useTheme();

  return (
    <div>
      <p>Current theme: {theme}</p>
      <button onClick={() => setTheme('dark')}>Switch to Dark</button>
      <button onClick={() => setTheme('light')}>Switch to Light</button>
      <button onClick={() => setTheme('system')}>Use System</button>
    </div>
  );
}
```

### Hook return values

| Value | Type | Description |
| --- | --- | --- |
| `theme` | `"light" \| "dark" \| "system"` | The currently selected theme. |
| `setTheme` | `(theme: Theme) => void` | Updates the theme and persists it. |

## ModeToggle

The library ships a ready-made `ModeToggle` dropdown that lets users switch themes:

```tsx
import { ModeToggle } from '@clidey/ux';

function Header() {
  return (
    <header className="flex items-center justify-between p-4">
      <h1>My App</h1>
      <ModeToggle />
    </header>
  );
}
```

`ModeToggle` renders a button with a sun/moon icon. Clicking it opens a dropdown with Light, Dark, and System options.

## How it works

`ThemeProvider` applies a CSS class to the `<html>` element:

- **Light mode** → no class (or `light` class)
- **Dark mode** → `dark` class added to `<html>`
- **System** → reads `prefers-color-scheme` media query and applies accordingly

All Clidey UX components use Tailwind's `dark:` modifier, so they respond automatically to the `dark` class on `<html>`.

## Customizing CSS variables

The stylesheet exposes CSS custom properties using OKLCH color values. Override them in your own CSS after importing `@clidey/ux/styles.css`:

```css
:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --primary: oklch(0.32 0.06 282.5);
  --primary-foreground: oklch(0.99 0 0);
  --secondary: oklch(0.97 0 0);
  --secondary-foreground: oklch(0.205 0 0);
  --muted: oklch(0.97 0 0);
  --muted-foreground: oklch(0.44 0 0);
  --accent: oklch(0.97 0 0);
  --accent-foreground: oklch(0.205 0 0);
  --destructive: oklch(0.577 0.245 27.325);
  --border: oklch(0.922 0 0);
  --input: oklch(0.922 0 0);
  --ring: oklch(0.708 0 0);
  --radius: 0.625rem;
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --primary: oklch(0.922 0 0);
  --primary-foreground: oklch(0.205 0 0);
  /* ... dark mode overrides ... */
}
```

### Backdrop token

Overlay components (`Dialog`, `AlertDialog`, `Sheet`, `Drawer`) use the `--backdrop` token for their semi-transparent overlay background instead of a hardcoded color:

```css
:root {
  --backdrop: oklch(0 0 0 / 50%);
}

.dark {
  --backdrop: oklch(0 0 0 / 60%);
}
```

Override `--backdrop` to customize overlay dimming across all modal components at once.

<Tip>
  Values use the OKLCH color space for perceptual uniformity. The format is `oklch(lightness chroma hue)` where lightness is 0-1, chroma is the saturation intensity, and hue is degrees. Achromatic colors use `oklch(L 0 0)`.
</Tip>
